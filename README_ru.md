# Musician's Canvas

Это многодорожечное приложение для записи музыки для настольных ПК. Оно написано на C++ и использует Qt в качестве
библиотеки графического интерфейса. Это приложение предназначено для Linux, Windows и Mac OS. Удобство использования является
основным критерием проектирования. Этот проект начался как эксперимент с Cursor (инструмент программирования с ИИ), чтобы увидеть, что
он сможет создать в качестве подобного приложения, с продолжением использования ИИ-разработки, а также
традиционной разработки программного обеспечения.

Много лет назад я создал программу многодорожечной записи музыки в университете (с тем же названием); это попытка
сделать что-то лучше.

## Возможности

- **Многодорожечная аранжировка**: Добавление и удаление нескольких дорожек для сочинения песен
- **Именованные дорожки**: Каждая дорожка имеет редактируемое имя, используемое как имя файла для записанного аудио
- **Типы дорожек**: Настройка каждой дорожки для аудиозаписи (с микрофона/линейного входа) или MIDI-записи (с клавиатуры/контроллера); нажмите на значок типа дорожки для быстрого изменения источника входа
- **Подготовка дорожки к записи**: Отметьте «Arm» на дорожке, чтобы выбрать её в качестве цели записи; одновременно может быть подготовлена только одна дорожка. Единственная незаписанная дорожка автоматически подготавливается для удобства
- **Обратный отсчёт записи**: 3-секундный визуальный обратный отсчёт перед началом записи, дающий исполнителю время на подготовку
- **Запись наложением (overdub)**: При записи новой дорожки, когда существующие дорожки включены, существующие дорожки микшируются и воспроизводятся в реальном времени, чтобы вы могли слышать их во время записи. Воспроизведение и захват синхронизированы для поддержания выравнивания всех дорожек
- **Визуальная обратная связь**: Отображение звуковой волны для аудиодорожек (с индикатором уровня в реальном времени во время записи), MIDI-представление пианоролл для MIDI-дорожек
- **Встроенный MIDI-синтезатор**: Рендерит MIDI-дорожки в аудио с помощью FluidSynth с настраиваемым SoundFont
- **Сведение в один аудиофайл**: Экспорт всех включённых дорожек в один смикшированный WAV или FLAC файл с использованием библиотеки [audio_mixer_cpp](https://github.com/EricOulashin/audio_mixer_cpp)
- **Сохранение / Открытие проекта**: Сериализация и восстановление полного проекта (дорожки, имена, типы, MIDI-ноты, ссылки на аудиофайлы) в/из JSON-файла с обнаружением несохранённых изменений при выходе
- **Настройки конкретного проекта**: Переопределение глобальных настроек MIDI и аудио для каждого проекта (частота дискретизации, SoundFont, MIDI-устройство)
- **Захват через PortAudio (опциональная сборка)**: Когда проект собран с PortAudio (`HAVE_PORTAUDIO`), запись может использовать нативный входной путь PortAudio (по духу аналогично Audacity) вместо Qt Multimedia. **Project → Project Settings → Audio** позволяет выбрать **PortAudio** или **Qt Multimedia** и выбрать входное устройство PortAudio. Если PortAudio не установлен, сборка всё равно проходит успешно, и запись использует только Qt Multimedia.
- **Высококачественное преобразование частоты дискретизации**: Записывает на нативной частоте аудиоустройства и преобразует к частоте проекта с помощью оконной sinc-интерполяции (окно Кайзера, ~96 дБ подавления в полосе заграждения) — то же семейство алгоритмов, что используется в Audacity / libsoxr. Это позволяет записывать на любой частоте дискретизации проекта (от 8000 Гц до 192000 Гц) независимо от нативной частоты устройства, без изменения высоты тона или длительности.
- **Автоматическая обработка моно/стерео**: Обнаруживает физически моно устройства, представляющиеся как стерео (часто встречается у USB-микрофонов веб-камер в PipeWire), и преобразует между моно и стерео по мере необходимости (дублирование или усреднение), следуя подходу маршрутизации каналов Audacity
- **Аудио с низкой задержкой**: В Windows обнаружение драйвера ASIO обеспечивает аудио с низкой задержкой; в Linux повышается приоритет планирования процесса для снижения джиттера с PipeWire / PulseAudio / ALSA
- **Virtual MIDI Keyboard**: Сопутствующее приложение для отправки MIDI-нот через программную клавиатуру фортепиано, со встроенным синтезатором FluidSynth, регулируемым мастер-усилением, отображением компьютерной клавиатуры на фортепиано, выбором инструмента/программы, управлением хорусом/эффектами и сдвигом октавы
- **Конфигурация**: Выбор устройства аудиовхода, MIDI-устройства и файла SoundFont (глобальные настройки по умолчанию и переопределения для каждого проекта)
- **Тёмная / светлая тема**: Настраивается через Settings → Configuration
- **Локализация**: Оба приложения локализованы на 18 языков: английский, немецкий, испанский, французский, японский, португальский (Бразилия), традиционный китайский, упрощённый китайский, русский, шведский, финский, датский, норвежский, польский, греческий, ирландский, валлийский и пиратский. Язык по умолчанию определяется настройками операционной системы и может быть изменён в **Settings → Configuration → Language**. Используется стандартная система i18n Qt (`QTranslator` / `.ts` / `.qm`), и интерфейс обновляется немедленно при смене языка.
- **Руководство пользователя**: HTML- и PDF-документация, сгенерированная из Markdown-исходников, доступна на всех поддерживаемых языках (см. [docs/](docs/))

## Зависимости

### Компилятор

Требуется компилятор с поддержкой **C++17** (GCC 8+, Clang 7+, MSVC 2017+).

---

### audio_mixer_cpp (требуется на всех платформах)

Функциональность микширования и экспорта FLAC зависит от библиотеки
[audio_mixer_cpp](https://github.com/EricOulashin/audio_mixer_cpp).
Клонируйте её как каталог-сосед этого репозитория перед сборкой:

```bash
git clone https://github.com/EricOulashin/audio_mixer_cpp.git
```

Сборка CMake ожидает найти её по пути `../audio_mixer_cpp` относительно корня этого проекта.

---

### Qt6 Multimedia (опционально, но настоятельно рекомендуется)

`qt6-multimedia` является опциональным. Без него приложение всё ещё собирается и работает, но
**аудиозапись и воспроизведение отключены** — будут работать только редактирование MIDI и управление проектами. Установите его вместе с основными библиотеками Qt6, используя приведённые ниже команды для соответствующей платформы.

---

### PortAudio (опционально)

Если библиотека и заголовочные файлы PortAudio установлены, CMake включает **`HAVE_PORTAUDIO`** и
связывает его с **musicians_canvas**. Запись по умолчанию использует путь PortAudio, если
проект не настроен на **Qt Multimedia** в **Project → Project Settings → Audio**. Если PortAudio
не найден, сборка продолжается без него (обёртка компилируется как заглушки).

---

### SoundFont для MIDI-синтеза

Для воспроизведения звука MIDI-дорожками требуется файл SoundFont (`.sf2`). Без него
MIDI-дорожки будут рендериться как тишина. В **Linux** встроенный синтезатор FluidSynth
автоматически обнаружит SoundFont, если он установлен по стандартному системному пути
(см. имена пакетов ниже). В **macOS и Windows** стандартного системного пути нет,
поэтому необходимо настроить SoundFont вручную в
**Settings → Configuration → MIDI**.

---

### Linux — Ubuntu / Debian

```bash
sudo apt install build-essential cmake \
  qt6-base-dev qt6-multimedia-dev \
  qt6-l10n-tools \
  libfluidsynth-dev librtmidi-dev libflac-dev \
  libportaudio2 portaudio19-dev \
  libpipewire-0.3-dev \
  fluid-soundfont-gm
```

> `fluid-soundfont-gm` устанавливает `FluidR3_GM.sf2` в `/usr/share/sounds/sf2/` и
> автоматически обнаруживается при запуске. `timgm6mb-soundfont` — меньшая альтернатива.
>
> `libpipewire-0.3-dev` требуется в системах на основе PipeWire, чтобы Virtual MIDI
> Keyboard мог вызвать `pw_init()` до создания FluidSynth своих аудиоресурсов.
> Сборка продолжается без него; флаг `HAVE_PIPEWIRE` просто не определяется.

### Linux — Fedora

```bash
sudo dnf install cmake gcc-c++ \
  qt6-qtbase-devel qt6-qtmultimedia-devel \
  fluidsynth-devel rtmidi-devel flac-devel \
  portaudio-devel \
  pipewire-devel \
  fluid-soundfont-gm
```

> `fluid-soundfont-gm` устанавливает `FluidR3_GM.sf2` в `/usr/share/soundfonts/` и
> автоматически обнаруживается при запуске.

### Linux — Arch Linux

```bash
sudo pacman -S base-devel cmake \
  qt6-base qt6-multimedia \
  fluidsynth rtmidi flac portaudio \
  pipewire \
  soundfont-fluid
```

> `soundfont-fluid` устанавливает `FluidR3_GM.sf2` в `/usr/share/soundfonts/` и
> автоматически обнаруживается при запуске. `pipewire` обычно уже установлен в современных
> системах Arch; заголовочные файлы для разработки включены в основной пакет.

### macOS

```bash
brew install cmake qt fluidsynth rtmidi flac portaudio
```

> PipeWire — это подсистема только для Linux и **не** требуется на macOS. FluidSynth
> автоматически будет использовать CoreAudio через бэкенд Qt Multimedia.
>
> Скачайте General MIDI SoundFont (например,
> [GeneralUser GS](https://schristiancollins.com/generaluser.php) или
> [FluidR3_GM](https://member.keymusician.com/Member/FluidR3_GM/index.html))
> и настройте его путь в **Settings → Configuration → MIDI**.

---

### Windows

**musicians_canvas** требует **ASIO-аудиодрайвер** для записи и воспроизведения с низкой задержкой.
Приложение не запустится, если не обнаружен ASIO-драйвер.
**virtual_midi_keyboard** не требует ASIO и использует бэкенд WASAPI Qt Multimedia напрямую.

**Установка ASIO-драйвера для musicians_canvas (выберите один):**

| Драйвер | Примечания |
|---------|------------|
| [ASIO4ALL](https://asio4all.org/) | Бесплатный, работает с большинством встроенного и USB-аудиооборудования |
| Драйвер производителя | Лучшая задержка для специализированных аудиоинтерфейсов (например, Focusrite, PreSonus, RME) |

**Инструментарий — MSYS2 (рекомендуется для сборок MinGW):**

Скачайте и установите MSYS2 с <https://www.msys2.org>, затем добавьте
`C:\msys64\mingw64\bin` в системный `PATH`. Установите необходимые пакеты:

```bash
pacman -S mingw-w64-x86_64-qt6-base
pacman -S mingw-w64-x86_64-qt6-multimedia
pacman -S mingw-w64-x86_64-fluidsynth
pacman -S mingw-w64-ucrt-x86_64-rtmidi
pacman -S mingw-w64-x86_64-flac
pacman -S mingw-w64-x86_64-portaudio
pacman -S mingw-w64-x86_64-soundfont-fluid
```

> `mingw-w64-x86_64-soundfont-fluid` устанавливает `FluidR3_GM.sf2` в
> `C:\msys64\mingw64\share\soundfonts\` (скорректируйте, если MSYS2 установлен в другом месте).
> Поскольку в Windows нет стандартного пути поиска SoundFont, необходимо настроить этот путь
> вручную в **Settings → Configuration → MIDI** после первого запуска.
>
> PipeWire — это подсистема только для Linux; пакет PipeWire на Windows не нужен.

Справочные страницы пакетов:
- <https://packages.msys2.org/packages/mingw-w64-x86_64-fluidsynth>
- <https://packages.msys2.org/packages/mingw-w64-ucrt-x86_64-rtmidi>
- <https://packages.msys2.org/packages/mingw-w64-x86_64-soundfont-fluid>

**Инструментарий — Visual Studio 2022:**

Установите Qt 6 через [Qt Online Installer](https://www.qt.io/download) и получите
бинарные файлы FluidSynth, RtMidi и libFLAC (или соберите их из исходного кода).
Сборка CMake найдёт их через `find_library` / `find_path`, если
соответствующие каталоги находятся в `CMAKE_PREFIX_PATH`.

SoundFont необходимо скачать отдельно (например,
[GeneralUser GS](https://schristiancollins.com/generaluser.php) или
[FluidR3_GM](https://member.keymusician.com/Member/FluidR3_GM/index.html)), и
настроить его путь в **Settings → Configuration → MIDI** после первого запуска.

> **Примечание:** `advapi32` (реестр Windows) и `winmm` (Windows Multimedia)
> автоматически связываются CMake; дополнительная установка для них не требуется.

## Сборка

```bash
mkdir build && cd build
cmake ..
make -j$(nproc)
```

**Windows (Visual Studio 2022):**

```cmd
mkdir build
cd build
cmake .. -G "Visual Studio 17 2022" -A x64
cmake --build . --config Release
```

Если необходимые DLL не найдены во время выполнения, запустите вспомогательный скрипт из каталога вывода сборки:

```cmd
..\..\copyRequiredWinDLLs.bat
```

## Использование

### musicians_canvas

1. **Установить каталог проекта**: Введите или перейдите к папке, где будут храниться записи и файл проекта
2. **Добавить дорожки**: Нажмите «+ Add Track»; назовите каждую дорожку в текстовом поле рядом с «Options»
3. **Настроить тип дорожки**: Нажмите «Options» на дорожке (или нажмите на значок типа дорожки между «Options» и полем имени), чтобы выбрать Audio или MIDI и установить источник входа
4. **Удалить дорожку**: Нажмите кнопку «×» в правой части строки дорожки
5. **Глобальные настройки**: Используйте **Settings → Configuration** для установки глобальных значений по умолчанию:
   - Вкладка *General*: Тема (тёмная/светлая)
   - Вкладка *MIDI*: MIDI-устройство вывода по умолчанию (встроенный синтезатор FluidSynth или внешнее MIDI-устройство) и SoundFont по умолчанию (`.sf2`) для синтеза
   - Вкладка *Audio*: Устройство аудиовхода/выхода по умолчанию для записи и воспроизведения
6. **Настройки проекта**: Используйте **Project → Project Settings** (Ctrl+P) для переопределения настроек MIDI и аудио только для текущего проекта (например, другая частота дискретизации или SoundFont для каждой песни)
7. **Запись**: Отметьте «Arm» на целевой дорожке, затем нажмите кнопку записи (красный круг). Одновременно может быть подготовлена только одна дорожка
8. **Воспроизведение**: Нажмите кнопку воспроизведения для микширования и воспроизведения всех включённых дорожек
9. **Сведение в файл**: Используйте **Tools → Mix tracks to file** (Ctrl+M) для экспорта в WAV или FLAC
10. **Сохранить / Открыть**: Используйте **File → Save Project** (Ctrl+S) и **File → Open Project** (Ctrl+O)

### virtual_midi_keyboard

1. **Открыть конфигурацию**: Используйте кнопку или меню **Configuration** (Ctrl+,) для открытия диалога настроек
2. **Вкладка MIDI**:
   - Выберите MIDI-устройство вывода (внешний аппаратный/программный синтезатор) или оставьте пустым для использования встроенного синтезатора FluidSynth
   - Выберите MIDI-устройство ввода для пересылки входящих MIDI-нот на дисплей клавиатуры
   - Отрегулируйте **Synthesizer Volume (Master Gain)** для управления уровнем выхода встроенного синтезатора (10%–200%)
3. **Вкладка Audio**: Выберите аудиоустройство вывода, используемое встроенным синтезатором
4. **SoundFont**: Выберите файл SoundFont `.sf2` для встроенного синтезатора (автоматически обнаруживается в Linux, если установлен системный SoundFont)
5. **Игра нот**: Нажимайте клавиши экранной клавиатуры фортепиано, используйте подключённый MIDI-контроллер или используйте компьютерную клавиатуру:
   - Нижняя октава: Z/X/C/V/B/N/M = C/D/E/F/G/A/B, S/D/G/H/J = диезы/бемоли
   - Верхняя октава: Q/W/E/R/T/Y/U/I/O/P = от C до E, 2/3/5/6/7/9/0 = диезы/бемоли
6. **Элементы управления панели инструментов**: Регулировка громкости MIDI (0–127), октавы (-3 до +5), уровня хоруса/эффекта и выбор инструментов

## Сочетания клавиш

**musicians_canvas:**

| Сочетание | Действие |
|-----------|----------|
| Ctrl+S | Сохранить проект |
| Ctrl+O | Открыть проект |
| Ctrl+M | Свести дорожки в файл |
| Ctrl+P | Настройки проекта |
| Ctrl+, | Настройки / Конфигурация |
| Ctrl+Q / Alt+F4 | Выход |

**virtual_midi_keyboard:**
